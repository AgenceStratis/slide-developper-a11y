ARIA

ARIA est une spécification technique du W3C qui a pour but d'améliorer l'accessibilité des contenus développés avec HTML, JavaScript, et les technologies connexes.

ARIA permet d'ajouter de la sémantique et des informations structurelles et comportementales, par l'utilisation de rôles, de propriétés et d'états aux éléments HTML.

  • Fournit aux aides techniques la sémantique des balises HTML, et notamment HTML5, sans utiliser les balises HTML natives.
  • Permet le support de HTML5 aux navigateurs et aides techniques anciens.
  • Peut étendre la sémantique au-delà de ce que propose HTML.
  • Peut changer le rôle, l'état et les propriétés des éléments HTML restitués par les aides techniques.

ARIA

Quand utiliser ARIA ?

ARIA ne doit être utilisé que lorsque c'est nécessaire !

Utilisez les fonctionnalités HTML natives en priorité pour fournir la sémantique.

HTML peut parfois montrer certaines limites : contrôle limité sur le code, composant trop complexe, équivalents HTML trop complexe à implémenter... ARIA devient alors un outil précieux pour améliorer l'accessibilité.

ARIA

Les attributs aria-*

Quelques attributs de zone Live

  • Indique que l'élément sera modifié afin que les technologies d'assistance puisse "l'écouter" : aria-live
    • Avec la valeur polite, les technologies d'assistance informeront l'utilisateur sans interrompre la tâche en cours (message à priorité peu élevée).
    • Avec la valeur assertive, les technologies d'assistance avertiront immédiatement l'utilisateur pouvant effacer la file d'attente des informations non-restituées (message à priorité très élevée).
  • Indique si les technologies d'assistance présenteront la totalité (true) ou seulement certaines parties de l'élément modifié ou seulement l'enfant modifié (false) : aria-atomic
  • Indique qu'un élément est en cours de modification (true) et que les technologies d'assistance doivent attendre que les modifications soient terminées avant de les exposer à l'utilisateur (false) : aria-busy
  • ...

Ressources : tous les attributs aria de zones Live dans les WCAG

ARIA

Les attributs aria-*

Quelques attributs de relation

  • Indique le ou les éléments dont la présence est contrôlée par l'élément courant : aria-controls
  • Définir une relation enfant/parent lorsque la hiérarchie du DOM ne peut pas être utilisée pour représenter la relation : aria-owns
  • Indique le ou les éléments qui identifient l'élément actuel : aria-labelledby
  • Indique le ou les éléments qui décrivent l'élément actuel : aria-describedby
  • Indique l'élément actuel comme étant un message d'erreur : aria-errormessage
  • ...

Les images

Deux types d'images

  • Les images de décoration
  • Les images porteuses d'informations

Attention : Les images dans les liens ou des boutons sont traitées comme des éléments interactifs, c'est donc la fonction et la destination qui doivent être restituées (voir la section liens et scripts).

Les images

Enjeu pour l'utilisateur

  • Une problématique importante pour les personnes aveugles et les grands malvoyants.
  • Apporter une alternative aux images porteuses d'informations.
  • Ne pas renseigner une alternative pour les images décoratives.
  • Ne pas créer de la redondance d'information.

Les images

Prendre en compte le contexte

Si le contexte est suffisamment pertinent et explicite, l'image devra être ignorée par les technologies d'assistance.

Astuce :
Cachez l'image et cherchez dans le contexte les informations véhiculées par l'image. Si ces informations sont présentes dans le contexte, pas besoin d'alternative à l'image.
Si les informations sont partielles ou absentes, alors il faudra ajouter une alternative à l'image.

Les images

Les images de décoration

Critère 1.2. Chaque image de décoration est-elle correctement ignorée par les technologies d'assistance ?

Les technologies d'assistance utilisées par les aveugles et grands malvoyants retranscrivent beaucoup d'information(type, propriétés, état, nom ou alternative).

Il est nécessaire de prendre en compte le contexte.

Dans le cas d'une image de décoration qui n'apporte pas d'information au contenu, l'image ne devra restituer aucune information à l'utilisateur.

Les images

Les images de décoration : la balise <img>

Pour ignorer une image <img> dans les technologies d'assistance,
au moins l'une de ces 2 conditions doit être vérifiée.

<img src="url.png" alt="">
  • L'attribut alt doit être vide. Ne pas mettre d'espace !
  • La balise <img> doit être dépourvue d'attribut title, aria-label, aria-labelledby. 
<img src="url.png" alt="alternative" aria-hidden="true">
  • La balise <img> possède un attribut aria-hidden="true".

Les images

Les images de décoration : la balise <svg>

Pour ignorer une image <svg> dans les technologies d'assistance,
toutes ces conditions doivent être vérifiées.

<svg aria-hidden="true" viewBox="0 0 512 512">
   <rect x="64.38" y="149.76" width="159.24" height="27.91"></rect>
</svg>
  • La balise <svg> doit posséder l'attribut aria-hidden="true".
  • Les balises enfants <title> et <desc> doivent être absentes.
  • La balise <svg> et ses enfants doivent être dépourvues d'alternative textuelle (attributs aria-label, aria-labelledby, aria-describedby).
  • La balise <svg> et ses enfants doivent être dépourvues d'attribut title.

Les images

Exemple

Le logo du département Seine et Marne en pied de page est une image de décoration, elle possède une alternative vide car le contexte est suffisant.

Les images qui véhiculent de l'information

Critère 1.1. Chaque image porteuse d'information a-t-elle une alternative textuelle ?

Critère 1.3. Pour chaque image porteuse d'information ayant une alternative textuelle, cette alternative est-elle pertinente (hors cas particuliers) ?

Il est nécessaire de prendre en compte le contexte.

Si l'information véhiculée par l'image est nécessaire à la compréhension, il est nécessaire de fournir une alternative.

  • Ne pas détailler l'ensemble des informations de l'image
  • Ne pas extrapoler une image, ni lui faire dire ce qu'elle ne dit pas
  • L'alternative doit être relativement courte
